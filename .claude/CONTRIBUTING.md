# 🤝 社区行为准则

保持尊重
所有社区交流均应遵循专业软件开发标准：
专业沟通
使用适合软件开发场景的清晰技术语言
提供具体、可操作的反馈并附上示例
聚焦技术价值与项目目标展开讨论
尊重不同的经验水平与学习方式
建设性协作
假定所有交流均出于善意
先提出澄清问题，再做出判断
为决策提供充分的背景信息与推理过程
认可有价值的贡献与乐于助人的社区成员
技术导向
讨论内容围绕软件开发与框架改进展开
基于技术价值、用户需求与项目定位做决策
用事实与示例支撑观点
注重实用价值而非理论完美
包容性环境
欢迎各技能水平与背景的贡献者
为新贡献者提供指导与帮助
通过代码评审与讨论创造学习机会
尊重多元的视角与解决方案

贡献类型

代码开发：框架功能、智能体、MCP 集成
文档编写：指南、示例、故障排查资源
测试验证：质量保障、边界场景发现、平台兼容性验证
社区建设：技术支持、方案分享、反馈建议、使用示例
金融行业专业知识：提供金融系统化软件开发的行业见解与需求分析
策略制定：参与项目规划、优先级设定与路线图讨论
知识传播：分享技术经验与趋势洞察
其他形式的贡献：如设计、营销、财务等
